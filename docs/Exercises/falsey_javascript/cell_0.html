In JavaScript, an empty array [] is considered a truthy value, meaning it's treated as true in a boolean context, resulting in unexpected comparisons when using the loose equality operator ==. This leads to [] == true returning true and [] == false returning false.

console.log([] == true)
console.log([] == false)

Comparison of Empty Array to Boolean Values

console.log([] == true)
console.log([] == false)

Explanation

  • In JavaScript, an empty array [] is considered a truthy value (i.e., it is treated as true in a boolean context).
  • The first console.log statement compares the empty array to true, which results in a coerced comparison due to JavaScript's loose equality operator. This coercion returns true.
  • The second console.log statement compares the empty array to false, which also results in a coerced comparison. This coercion returns false, because in a boolean context, an empty array is considered true, not false.
